World Tracker

Type: Project • Year: 2025 • Status: Completed

What this project does

World Tracker lets multiple users keep a personal log of the countries they’ve visited. Data is stored in PostgreSQL and rendered via EJS onto a world map. A user types a country name, the server validates it against the countries table (exact / starts‑with / contains using ILIKE), and if valid it gets saved to that user and the corresponding country is highlighted on the map.

  • Multiple users, each with their own visited_countries set
  • Server-side validation against a canonical countries table (exact ➝ begins-with ➝ contains via ILIKE)
  • Duplicate protection and clear feedback messages (errors/success)
  • Dynamic EJS rendering to color the visited countries and update totals

Tech stack

EJS CSS3 JavaScript Node.js Express PostgreSQL

Role & challenges

I built the full stack: EJS views, Express routes, and PostgreSQL queries. Key challenges included text matching for country inputs (balancing ILIKE strategies for exact vs. partial matches), keeping performance acceptable, and preventing duplicates per user (unique constraints/logic). I also implemented user switching and safe parameterized queries throughout.

What I learned

  • EJS templating patterns for conditional rendering and partials
  • Designing clean Express routes with server-side validation
  • Parameterized queries in PostgreSQL and basic indexing for text lookup
  • Conflict handling (ON CONFLICT DO NOTHING) and unique-per-user logic
