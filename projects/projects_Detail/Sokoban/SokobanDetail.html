Sokoban

Type: University Game • Year: 2023 • Status: Completed

What this project does

Sokoban is a classic puzzle game recreated as a pair project for the Object‑Oriented Programming course. It was built in Java using the Eclipse IDE. The player controls a forklift on a grid (matrix) and must push crates onto target cells. The board is modeled as layered matrices where each cell can contain terrain, interactive objects and entities.

  • Multiple levels with increasing difficulty
  • Layered tile system (terrain + objects + entity)
  • Special tiles & items: holes (you lose if you fall), damaged walls (breakable with hammers as catchables), pallets (to cover holes), portals, and batteries to recharge the forklift
  • Move counter & performance scoring (fewer moves is better)
  • Energy system: each move consumes battery; reaching zero battery results in a loss

Tech stack

Java Eclipse IDE Object‑Oriented Design 2D Grid Engine Level Parser

Role & challenges

Built in a pair, I focused on core mechanics and systems design: level parsing, movement & collision rules for pushing crates, layered tile rendering/updates, and the energy/battery system. Challenges included consistent rules when combining special tiles (e.g., pushing over holes covered by pallets, interactions with portals), and keeping the codebase modular and testable.

What I learned

  • Applying OOP principles (encapsulation, inheritance vs. composition)
  • Designing a simple game loop and deterministic movement rules
  • Implementing a layered grid model and collision handling
  • Parsing level files and validating states
  • Pair programming workflow and version control best practices
