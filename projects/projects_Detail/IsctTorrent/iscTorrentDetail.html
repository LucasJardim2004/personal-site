IscTorrent

Type: University Project • Year: 2024 • Status: Completed

What this project does

IscTorrent is a peer-to-peer (P2P) application developed in Java (IntelliJ) for the Concurrent and Distributed Programming course. Each user runs a local node (e.g., ports 8080, 8081, …) and connects to other nodes in the network. When searching for a .mp3 song, the request is sent via broadcast to connected peers; any peer with the file responds and starts sending it concurrently in ordered byte blocks. Once all blocks arrive, they are reassembled on the client side, and the user now owns the song.

  • Peer discovery via broadcast on active connections
  • Parallel block transfers (multi-source) with reordering on the client
  • Thread pool to handle simultaneous uploads/downloads
  • Reassembled file persistence and basic integrity check

Tech stack

Java IntelliJ IDEA TCP/IP Sockets Multithreading Thread Pool P2P

Role & challenges

Project done in pairs. I designed and implemented the simple discovery/transfer protocol, block division, and coordination of uploads/downloads using a ThreadPool. Main challenges: synchronization and ordering of blocks coming from multiple sources, avoiding race conditions, and managing I/O without blocking the UI/flow.

What I learned

  • Programming with ServerSocket/Socket and messages between peers
  • Thread-safe structures and coordination with synchronized and ConcurrentHashMap
  • File division, ordering, and reassembly by blocks
  • Sizing ThreadPool and managing I/O-bound tasks
  • Testing on localhost and local network with multiple ports
