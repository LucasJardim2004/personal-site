GeoOrganizer

Type: University Project • Year: 2025 • Status: Completed

What this project does

GeoOrganizer is a desktop application built in Java with a Swing UI. Users upload a CSV file where each row represents a property/parcel. The app parses the file and builds a graph: each node is a property and edges connect adjacent properties, creating a clear visual network. For the demo, we used a dataset from Madeira Island.

  • CSV import with schema validation and error feedback
  • Graph visualization where nodes are properties and edges mean adjacency
  • Filters by district/island, municipality and parish, and by owner ID
  • Interactive highlight/zoom to explore specific owners or regions
  • Foundation for property-exchange suggestions between owners to normalize land distribution

Tech stack

Java Swing IntelliJ IDEA CSV Parsing Graph Model

Role & challenges

This project was developed in a group of four students for the Software Engineering course. The development was managed using Agile SCRUM practices, with regular sprints, backlog management, and team role distribution. My contribution was implementing the full technical flow: CSV ingestion, data validation, graph construction and the Swing UI. Challenges included keeping the UI responsive with larger datasets (EDT/threading), defining adjacency rules from CSV fields, and designing meaningful filters without overwhelming the interface.

What I learned

  • File I/O and CSV parsing patterns in Java
  • Modeling geographic data as graphs (nodes/edges) and rendering it
  • Swing best practices: Event Dispatch Thread, background tasks, and UI state
  • Designing filterable UIs for data exploration
  • First steps toward recommendation logic for property exchanges
